REPORTER: Cekura, a YC F24 startup, has launched with a compelling solution for a critical problem in the AI world: how to effectively test and monitor the behavioral regressions of AI agents. As AI agents become more complex, manual QA is simply not scalable, and traditional scripted tests often fall short. Cekura’s answer is simulation. They use synthetic users to interact with AI agents in a realistic conversational manner. LLM-based judges then evaluate whether the agent responded correctly across the full conversational arc, not just single turns.

Finally, Cekura focuses on deterministic, structured test cases. Instead of free-form prompts, their evaluators use structured conditional action trees with explicit conditions and support for fixed messages. This means synthetic user behavior is consistent across runs, making failures real regressions, not noise.

Beyond testing, Cekura also monitors live agent traffic. While tracing platforms debug individual LLM calls, Cekura evaluates the full session. This is critical because many AI agent failures aren't in a single turn, but in how turns relate. For example, if a banking agent skips a crucial verification step and proceeds, individual turns might look fine to a turn-based evaluator. Cekura’s judge sees the full transcript and flags the session as failed because the verification never occurred.
